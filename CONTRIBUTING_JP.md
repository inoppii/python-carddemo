# コントリビューションガイドライン

プロジェクトへの貢献に関心をお寄せいただきありがとうございます。バグレポート、新機能、修正、または追加のドキュメントのいずれであっても、コミュニティからのフィードバックと貢献を大いに尊重します。

バグレポートや貢献に効果的に対応するために必要なすべての情報が揃っていることを確認するために、課題やプルリクエストを送信する前にこのドキュメントをご一読ください。

## バグの報告/機能のリクエスト

バグの報告や機能の提案には、GitHub の issue tracker を使用してください。

issue を提出する前に、既存のオープンな issue や最近クローズされた issue を確認し、他の誰かがすでにその問題を報告していないことを確認してください。できるだけ多くの情報を含めるようにしてください。以下のような詳細は非常に役立ちます：

* 再現可能なテストケースまたは一連の手順
* 使用されているコードのバージョン
* バグに関連して行った修正
* 環境やデプロイに関する異常な点

## プルリクエストによる貢献

プルリクエストによる貢献は大歓迎です。プルリクエストを送信する前に、以下のことを確認してください：

1. *main* ブランチの最新のソースに対して作業していること。
2. 既存のオープンなプルリクエストや最近マージされたプルリクエストを確認し、他の誰かがすでにその問題に対処していないことを確認すること。
3. 重要な作業については issue をオープンして議論すること - あなたの時間が無駄になることは避けたいと考えています。

プルリクエストを送信するには、以下の手順に従ってください：

1. リポジトリをフォークします。
2. ソースを変更します。貢献する特定の変更に焦点を当ててください。すべてのコードを再フォーマットすると、変更点に焦点を当てることが難しくなります。
3. ローカルテストが通過することを確認します。
4. 明確なコミットメッセージを使用して、フォークにコミットします。
5. プルリクエストを送信し、プルリクエストインターフェイスのデフォルトの質問に回答します。
6. プルリクエストで報告された自動 CI の失敗に注意し、会話に参加し続けます。

GitHub は、[リポジトリのフォーク](https://help.github.com/articles/fork-a-repo/) および [プルリクエストの作成](https://help.github.com/articles/creating-a-pull-request/) に関する追加ドキュメントを提供しています。

## 取り組むべき貢献を見つける

既存の issue を見ることは、貢献すべきことを見つけるための素晴らしい方法です。私たちのプロジェクトは、デフォルトで GitHub の標準的な issue ラベル (enhancement/bug/duplicate/help wanted/invalid/question/wontfix) を使用しているため、「help wanted」の issue を見ることから始めるのが最適です。
